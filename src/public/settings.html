Enable structured logs & metrics
When on, every question / OCR run / latency measurement is sent to Grafana Cloud. CPU / GPU / memory gauges are sampled every 10 s.
Local fallback active — all events that would go to Grafana are being written to logs/telemetry_node.jsonl and logs/telemetry_python.jsonl instead. Both files are overwritten on every server restart.
Setup steps
1. Don't have a Grafana stack? Sign up free at grafana.com/auth/sign-up (10k metric series + 50 GB logs included; 14-day trial then free tier). Already have one? Open it from grafana.com/orgs.
2. In your Grafana stack, open the OpenTelemetry setup page: docs link. Or in your Grafana web UI: left sidebar → Connections → Add new connection → search "OpenTelemetry" → click OpenTelemetry (OTLP). Copy the endpoint URL + Instance ID (e.g. 1465941).
3. On the same page, click Generate now to create an access-policy token (or go to Administration → Access Policies → Create access policy; scopes metrics:write + logs:write). Copy the glc_… value.
4. Paste the three values below and click Validate & Enable.
Auto-import dashboard
One-click setup: creates the SolveWatch dashboard in your Grafana stack. Needs a separate Service Account token (different from the OTLP token above).
Steps:
  1. In your Grafana stack: left sidebar → Administration → Users and access → Service accounts (direct path: <your-grafana-url>/org/serviceaccounts)
  2. Use an existing service account, or click Add service account. Set role to Admin so it can write dashboards in any folder.
  3. Open the service account → Add service account token → name it (e.g. "SolveWatch importer") → click Generate token → copy the glsa_… value.
  4. Paste your stack URL + the token below and click Import dashboard to Grafana.
